GCC

选项 作用
-E 预处理,生成.i文件
-S 编译，生成.s文件
-c 汇编，生成.o文件
NULL 连接，生成可执行文件
-o 输出文件，以上均可用
-l(小写L) 后面跟要添加的lib库
-I(大写I) 后面跟要添加的头文件目录
-L 后面跟要添加库的目录
-fPIC -shared 打包成共享动态库

连接器的作用是连接目标代码，系统启动代码，库代码。

c++ 两阶段名字查找，对于非 dependent name 都是现场决议的，所以空写 f 的时候，编译器不查找基类 scope，没找到就报错，编译器不查找基类，因为此时基类还是个类模板，到底是什么东西还不定呢。
然后实例化的时候进行第二阶段名字查找，这时候所有类型都是确定的，才能够查找基类 scope。
增加 this 或 Y:: 就是把对 f 的查找延迟到第二阶段，这是 c++ 标准规定的。

MAKE

makefile实际上有作用的是命令行，它对文件依赖的生成并没有多大作用….

$^命令代表依赖文件，但它等于”找到的依赖文件的路径”,所以命令行里尽量使用他，因为makefile实际工作的是命令行，就算你找不找的到所依赖的文件，他都按你敲的命令行工作，所以…..，这个makefile还真是鸡肋…,，$@代表目标文件，仅此而以

makefile命令执行顺序，从上到下，从左到右

变量 含义
$@ 规则中的目标文件
$< 规则中第一个依赖文件,但是有隐藏规则，所以才可以用来遍历依赖文件
$?
$^ 规则中所有依赖文件
$(RM) rm -f
$(AR) ar

第一个（伪）目标总是默认目标，总是被执行。而且每个makefile只有一个总目标，如果你实质上要生成多个目标，你需要先用个总目标，否则它默认取一个目标，从而达不到你要的结果。

CMake

不同的编译工具(eg.nmake GNU make..)，有不同的语法，他们的工作方式都是读取一个文本文件(makefile)。正因为不同的编译工具是应用不同的语法，我们在不同的平台上就要写不同的makefile.cmake的工作就是自动查找当前平台下的编译工具，生成对应的文本文件(makefile)。

  • 改变安装目录 CMAKE_INSTALL_PREFIX
  • su进入root命令行

命令和变量 含义
PROJECT_SOURCE_DIR cmake文件所在目录
PROJECT_BINARY_DIR 执行cmake命令的目录
SET() 声明变量和赋值
MESSAGE（FATAL_ERROR “not found”） 终止程序，并打印信息。或为SEND_ERORR,或省略..
PROJECT(NAME) 工程名字
ADD_EXECUTABLE(hello main.c) 添加要生成的可执行程序
EXECUTABLE_OUTPUT_PATH 当前CMAKEFILE可执行程序的输出路径
LIBRARY_OUTPUT_PATH 当前CMAKEFILE库的输出目录
ADD_SUBDIRECTORY(dir) 执行dir的CMAKEFILE
CMAKE_INSTALL_PREFIX 为以下install命令设置目录
INSTALL(TARGETS target1 target2.. ARCHIVE DESTINATION dir1 LIBRARY DESTINATION dir2 RUNTIME DESTINATION dir3)
INSTALL（FILES file1 DESTINATION dir）
INSTALL(PROGRAMS file1 DESTINATION dir)
INSTALL(DIRECTORY dir1 DESTINATION)
ADD_LIBRARY(libname STATIC|SHARED|MODULE src.c ..) 生成共享库
SET_TARGET_PROPERTIES(libname PROPERTIES OUTPUT_NAME “real_out_name”) 设置目标的输出文件名字
SET_TARGET_PROPERTIES(libname PROPERTIES CLEAN_DIRECT_OUTPUT 1)
INCLUDE_DIRECTORIES（） 添加头文件目录（对当前CMAKEFILE所有目标有效）
LINK_DIRECTORIES() 添加文件目录（对当前CMAKEFILE所有目标有效）
TARGET_INCLUDE_DIRECTORIES() 添加头文件目录只对当前target有效
LINK_LIBRARIES() 连接库文件
TARGET_LINK_LIBRARIES() 连接库文件，自己生成的库，且用来生成依赖关系
FIND_PATH(var filename) 找到filename并把包含路径的文件名赋给var
CMAKE_INCLUDE_PATH 环境变量
CMAKE_LIBRARY_PATH 环境变量
CMAKE_PREFIX_PATH 由用户设置,为所有find命令提供路径
CMAKE_MODULE_PATH 为include(),find_package()命令提供路径
FIND_PACKAGE(lib)
add_compile_options(-g -Wall -lstdc++) （对所有编译器）添加编译选项,SET(CMAKE_C_FLAGS)是只对C编译器
aux_source_directory(dir var) 搜索dir目录下的源文件并且赋给var
add_definition(-DLIBO) 添加宏LIBO，和gcc -d一样
CMAKE_ROOT cmake
FILE(WRITE filename “content”)
CMAKE_CURRENT_SOURCE_DIR 当前cmakefile路径

自己生成的库，且用target_link_libraries()添加依赖关系了，那么这个库cmake一定能找到，你不用为这个库添加库目录了

库的链接顺序会导致编译错误，比如在CMake中 target_link_libraries(main viewer_matrix4x4 viewer_openglalg)

的链接顺序会导致符号函数找不到。

可以使用target_link_libraries(main -Wl,–start-group viewer_matrix4x4 viewer_openglalg -Wl,–end-group)(gcc也有相应的命令)
