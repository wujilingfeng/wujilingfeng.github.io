1. 从连续能量到弱形式，再到离散系统
你写的连续能量是
$E(\chi)=\int_\Omega \|\nabla \chi - V\|^2\,dx.$
这里 $\chi$ 是未知标量函数，$V$ 是已知向量场。
Poisson 重建里，$\chi$ 通常就是“指示函数”或其平滑版本，$\nabla\chi$ 希望尽量贴近由法向点云估计出来的向量场 $V$。

1.1 为什么最小化这个能量会得到 Poisson 方程
把能量展开：
$E(\chi)=\int_\Omega \left( \nabla\chi\cdot\nabla\chi - 2V\cdot\nabla\chi + V\cdot V \right)dx.$
其中最后一项 $\int_\Omega V\cdot V\,dx$ 与 $\chi$ 无关，所以最小化等价于最小化
$\widetilde E(\chi)=\int_\Omega \left( \nabla\chi\cdot\nabla\chi - 2V\cdot\nabla\chi \right)dx.$

1.2 变分推导弱形式
令任意测试函数 $\phi$，考虑扰动
$\chi_\varepsilon=\chi+\varepsilon\phi.$
把它代入能量：
$E(\chi+\varepsilon\phi)=\int_\Omega \|\nabla\chi+\varepsilon\nabla\phi -V\|^2 dx.$
对 $\varepsilon$ 求导并令 $\varepsilon=0$：
$\frac{d}{d\varepsilon}E(\chi+\varepsilon\phi)\Big|_{\varepsilon=0}
=
2\int_\Omega (\nabla\chi -V)\cdot \nabla\phi\,dx.$
极值条件要求对所有允许的 $\phi$ 都有
$\int_\Omega (\nabla\chi -V)\cdot \nabla\phi\,dx=0,$
即
$\int_\Omega \nabla\chi\cdot\nabla\phi\,dx
=
\int_\Omega V\cdot\nabla\phi\,dx.$
这就是你写的弱形式。

1.3 弱形式如何变成强形式
从
$\int_\Omega \nabla\chi\cdot\nabla\phi\,dx
=
\int_\Omega V\cdot\nabla\phi\,dx$
出发，对右边和左边都做分部积分。
先看左边：
$\int_\Omega \nabla\chi\cdot\nabla\phi\,dx
=
-\int_\Omega (\Delta\chi)\phi\,dx
+\int_{\partial\Omega}\frac{\partial\chi}{\partial n}\phi\,ds.$
再看右边：
$\int_\Omega V\cdot\nabla\phi\,dx
=
-\int_\Omega (\nabla\cdot V)\phi\,dx
+\int_{\partial\Omega}(V\cdot n)\phi\,ds.$
于是得到
$-\int_\Omega (\Delta\chi)\phi\,dx
+\int_{\partial\Omega}\frac{\partial\chi}{\partial n}\phi\,ds
=
-\int_\Omega (\nabla\cdot V)\phi\,dx
+\int_{\partial\Omega}(V\cdot n)\phi\,ds.$
整理：
$\int_\Omega (\Delta\chi-\nabla\cdot V)\phi\,dx
=
\int_{\partial\Omega}\left(\frac{\partial\chi}{\partial n}-V\cdot n\right)\phi\,ds.$
如果对所有测试函数 $\phi$ 都成立，那么在足够光滑时得到强形式：
$\Delta\chi=\nabla\cdot V \qquad \text{in }\Omega,$
边界自然条件为
$\frac{\partial\chi}{\partial n}=V\cdot n \qquad \text{on }\partial\Omega.$
所以：


弱形式是“最小化能量的一阶最优条件”


强形式是把弱形式再通过分部积分写成 PDE



2. 为什么离散后就是 $Ax=b$
这本质上是有限元/Galerkin 离散。

2.1 用有限维空间近似未知函数
选一个有限维函数空间 $X_h\subset H^1(\Omega)$，基函数为 $\{B_\alpha\}$。
把未知函数写成
$\chi_h(x)=\sum_\alpha x_\alpha B_\alpha(x).$
其中 $x_\alpha$ 是待求系数。
测试函数也取自同一个空间（Galerkin）：
$\phi_h = B_\beta.$

2.2 把弱形式限制到有限维空间
原弱形式是
$\int_\Omega \nabla\chi\cdot\nabla\phi\,dx
=
\int_\Omega V\cdot\nabla\phi\,dx.$
现在要求它只对所有 $\phi_h\in X_h$ 成立：
$\int_\Omega \nabla\chi_h\cdot\nabla\phi_h\,dx
=
\int_\Omega V_h\cdot\nabla\phi_h\,dx.$
把 $\chi_h=\sum_\alpha x_\alpha B_\alpha$ 代进去：
$\int_\Omega \nabla\left(\sum_\alpha x_\alpha B_\alpha\right)\cdot\nabla B_\beta\,dx
=
\int_\Omega V_h\cdot\nabla B_\beta\,dx.$
线性性给出
$\sum_\alpha x_\alpha \int_\Omega \nabla B_\alpha\cdot\nabla B_\beta\,dx
=
\int_\Omega V_h\cdot\nabla B_\beta\,dx.$
于是定义
$A_{\beta\alpha}=\int_\Omega \nabla B_\alpha\cdot\nabla B_\beta\,dx,
\qquad
b_\beta=\int_\Omega V_h\cdot\nabla B_\beta\,dx,$
就得到
$\sum_\alpha A_{\beta\alpha}x_\alpha=b_\beta.$
写成矩阵形式就是
$Ax=b.$
这正是你给出的离散系统。

2.3 这和“直接离散 PDE”为什么一致
因为你离散的不是随便来的，而是弱形式的 Ritz/Galerkin 投影。
在这个框架下，离散解 $\chi_h$ 是在 $X_h$ 中最小化离散能量的人：
$E_h(\chi_h)=\int_\Omega \|\nabla\chi_h -V_h\|^2 dx.$
把 $\chi_h=\sum_\alpha x_\alpha B_\alpha$ 代进去，这就是一个关于系数 $x$ 的二次型：
$E_h(x)=x^T A x - 2x^T b + c.$
对 $x$ 求导：
$\nabla_x E_h = 2Ax - 2b.$
令其为零：
$Ax=b.$
所以：


连续能量最小化


弱形式


有限元离散后的线性系统


三者是完全同一件事在不同层级的表达。

3. 边界条件带进去后，连续和离散怎么写
边界条件分三类最常见：Dirichlet、Neumann、Robin/混合。

3.1 Dirichlet 边界条件
设
$\chi=g \qquad \text{on }\partial\Omega_D.$
这时允许的解空间是
$\mathcal U_g=\{\chi\in H^1(\Omega):\chi|_{\partial\Omega_D}=g\},$
测试函数必须满足
$\phi=0 \qquad \text{on }\partial\Omega_D.$
弱形式变成：
$\int_\Omega \nabla\chi\cdot\nabla\phi\,dx
=
\int_\Omega V\cdot\nabla\phi\,dx,
\qquad \forall \phi\in H^1_0(\Omega;\partial\Omega_D).$
此时分部积分时，$\phi$ 在 $\partial\Omega_D$ 上为零，所以边界项消失。
如果剩余边界 $\partial\Omega_N$ 没有显式指定，就得到自然 Neumann 条件
$\frac{\partial\chi}{\partial n}=V\cdot n
\qquad \text{on }\partial\Omega_N.$

Dirichlet 的离散处理
如果某些基函数自由度在边界上被固定，例如节点 $i\in D$ 上
$x_i=g_i,$
则把未知量分成自由部分 $x_F$ 和已知边界部分 $x_D$。矩阵分块：
$\begin{bmatrix}
A_{FF} & A_{FD}\\
A_{DF} & A_{DD}
\end{bmatrix}
\begin{bmatrix}
x_F\\
x_D
\end{bmatrix}
=
\begin{bmatrix}
b_F\\
b_D
\end{bmatrix}.$
因为 $x_D$ 已知，实际求解
$A_{FF}x_F=b_F-A_{FD}x_D.$
这就是有限元里最标准的 Dirichlet 处理。
工程上也常直接“改行改列”强制边界值。

3.2 Neumann 边界条件
如果给定
$\frac{\partial \chi}{\partial n}=q \qquad \text{on }\partial\Omega,$
那么对应的强形式若从标准 Poisson
$-\Delta \chi = f$
出发，弱形式一般是
$\int_\Omega \nabla\chi\cdot\nabla\phi\,dx
=
\int_\Omega f\phi\,dx+\int_{\partial\Omega} q\phi\,ds.$
而你这个问题原本来自能量 $\|\nabla\chi-V\|^2$，其自然边界就是
$\frac{\partial\chi}{\partial n}=V\cdot n.$
所以当你不显式加 Dirichlet 约束时，得到的是自然 Neumann 问题。

Neumann 的离散系统
若边界项存在，则右端项要加边界积分：
$b_\beta
=
\int_\Omega V_h\cdot\nabla B_\beta\,dx
+
\int_{\partial\Omega_N} t_h B_\beta\,ds$
或在标准 Poisson 记号下
$b_\beta
=
\int_\Omega f_h B_\beta\,dx
+
\int_{\partial\Omega_N} q_h B_\beta\,ds.$
注意：纯 Neumann 问题通常只确定到一个常数。
因为若 $\chi$ 是解，则 $\chi+C$ 也是解。于是矩阵 $A$ 会有零空间：
$A\mathbf 1 = 0$
（常数函数零梯度）。
这时需要加一个约束，例如：


固定一个点值


或要求平均值为零


$\int_\Omega \chi\,dx=0.$
Poisson 重建里经常就有这个“常数不定性”。

3.3 Robin 边界条件
若边界条件是
$\frac{\partial\chi}{\partial n}+\gamma\chi=r
\qquad \text{on }\partial\Omega_R,$
则弱形式中会多一个边界双线性项：
$\int_\Omega \nabla\chi\cdot\nabla\phi\,dx
+
\int_{\partial\Omega_R}\gamma\chi\phi\,ds
=
\int_\Omega f\phi\,dx
+
\int_{\partial\Omega_R}r\phi\,ds.$
离散后矩阵和右端分别多出
$A^{(R)}_{\alpha\beta}
=
\int_{\partial\Omega_R}\gamma B_\alpha B_\beta\,ds,
\qquad
b^{(R)}_\beta
=
\int_{\partial\Omega_R} r B_\beta\,ds.$

4. Poisson 表面重建到底在解什么

4.1 从点法向到向量场
输入是带法向的点云
$\{(p_i,n_i)\}_{i=1}^N.$
目标是恢复一个隐式函数 $\chi$，其等值面给出曲面。
Poisson 重建的核心想法是：


理想情况下，物体内部指示函数 $\chi$ 在表面附近发生跳变；


它的梯度 $\nabla\chi$ 会沿法向集中在表面附近；


所以如果我们从点法向构造一个向量场 $V$，就希望


$\nabla\chi \approx V.$
因此解
$\min_\chi \int_\Omega \|\nabla\chi -V\|^2 dx.$

4.2 为什么会得到 $\Delta\chi=\nabla\cdot V$
前面已经推了，直接得到
$\Delta\chi=\nabla\cdot V.$
这就是 Poisson 方程。
这里的“源项”不是普通标量 $f$，而是法向构造向量场的散度。

4.3 向量场 $V$ 在离散里如何构造
理论上，可以把点法向看成分布：
$V(x)\approx \sum_i n_i\,\delta(x-p_i).$
但这太奇异，不适合直接数值求解。
所以实际里会把点法向“泼洒/splat”到某个离散基上，得到平滑的 $V_h$。
若使用有限元基 $B_\alpha$，常见做法是把向量场也投影到某个向量值空间：
$V_h(x)=\sum_j v_j \Psi_j(x).$
然后右端项
$b_\beta=\int_\Omega V_h\cdot\nabla B_\beta\,dx.$
在 PoissonRecon 那类实现里，常常根本不显式形成连续 $V_h$ 再积分，而是直接把点法向对右端贡献累加到相邻基函数支持域中。

5. Screened Poisson 重建：理论和离散推导
screened Poisson 的核心是：
普通 Poisson 只约束梯度匹配，低频部分尤其常数/大尺度漂移不够稳；
所以再加一个“函数值采样约束”，让 $\chi$ 在采样点附近更贴合某个目标值。

5.1 screened 的连续能量
一种典型形式是
$E(\chi)
=
\int_\Omega \|\nabla\chi -V\|^2 dx
+
\lambda \int_\Omega w(x)\,(\chi-\chi_0)^2 dx.$
其中：


$V$：由法向构造的向量场


$\chi_0$：希望贴近的函数值


$w(x)$：权重，常常集中在采样点附近


$\lambda>0$：screening 强度


如果采样点是 $\{p_i\}$，也常写成离散采样项：
$E(\chi)
=
\int_\Omega \|\nabla\chi -V\|^2 dx
+
\lambda \sum_i \omega_i\big(\chi(p_i)-c_i\big)^2.$
这是最常见、最直观的 screened 形式。
在表面重建里，常把 $c_i$ 设为某个等值面值，例如 0 或 1/2，意思是“采样点应落在重建隐式函数的表面上”。

5.2 screened 的弱形式推导
先看连续权重形式：
$E(\chi)=
\int_\Omega \|\nabla\chi -V\|^2 dx
+
\lambda \int_\Omega w(\chi-\chi_0)^2 dx.$
对 $\chi+\varepsilon\phi$ 求一阶变分：
$0=
\frac{d}{d\varepsilon}E(\chi+\varepsilon\phi)\Big|_{\varepsilon=0}
=
2\int_\Omega (\nabla\chi -V)\cdot\nabla\phi\,dx
+
2\lambda\int_\Omega w(\chi-\chi_0)\phi\,dx.$
因此弱形式为
$\int_\Omega \nabla\chi\cdot\nabla\phi\,dx
+
\lambda\int_\Omega w\,\chi\,\phi\,dx
=
\int_\Omega V\cdot\nabla\phi\,dx
+
\lambda\int_\Omega w\,\chi_0\,\phi\,dx.$

5.3 screened 的强形式
对第一项分部积分，可以写成
$-\Delta\chi + \lambda w \chi
=
-\nabla\cdot V + \lambda w\chi_0$
如果你采用
$\Delta\chi=\nabla\cdot V$
这种号约定，那么 screened 版本对应写成
$\Delta\chi - \lambda w\chi
=
\nabla\cdot V - \lambda w\chi_0.$
这只是符号约定问题，本质上都是“Laplacian + mass term”。
更标准的有限元写法一般是
$(\nabla\chi,\nabla\phi)+\lambda (w\chi,\phi)
=
(V,\nabla\phi)+\lambda (w\chi_0,\phi).$

5.4 点采样型 screened 项
如果 screening 是点采样
$E(\chi)=
\int_\Omega \|\nabla\chi -V\|^2 dx
+
\lambda\sum_i \omega_i(\chi(p_i)-c_i)^2,$
则变分给出
$\int_\Omega \nabla\chi\cdot\nabla\phi\,dx
+
\lambda\sum_i \omega_i \chi(p_i)\phi(p_i)
=
\int_\Omega V\cdot\nabla\phi\,dx
+
\lambda\sum_i \omega_i c_i\phi(p_i).$
这非常适合解释 screened PoissonRecon 的离散结构。

5.5 screened 的离散系统
令
$\chi_h=\sum_\alpha x_\alpha B_\alpha.$
代入弱形式，若是连续权重型，得到
$\sum_\alpha x_\alpha
\left[
\int_\Omega \nabla B_\alpha\cdot\nabla B_\beta\,dx
+
\lambda\int_\Omega w\,B_\alpha B_\beta\,dx
\right]
=
\int_\Omega V_h\cdot\nabla B_\beta\,dx
+
\lambda\int_\Omega w\,\chi_0 B_\beta\,dx.$
于是矩阵形式为
$(A+\lambda M_w)x=b+\lambda d,$
其中
$A_{\beta\alpha}=\int_\Omega \nabla B_\alpha\cdot\nabla B_\beta\,dx,$
$(M_w)_{\beta\alpha}=\int_\Omega w\,B_\alpha B_\beta\,dx,$
$b_\beta=\int_\Omega V_h\cdot\nabla B_\beta\,dx,
\qquad
d_\beta=\int_\Omega w\,\chi_0 B_\beta\,dx.$
如果是点采样型 screening，则
$\sum_\alpha x_\alpha
\left[
A_{\beta\alpha}
+
\lambda \sum_i \omega_i B_\alpha(p_i)B_\beta(p_i)
\right]
=
b_\beta
+
\lambda \sum_i \omega_i c_i B_\beta(p_i).$
定义
$S_{\beta\alpha}=\sum_i \omega_i B_\alpha(p_i)B_\beta(p_i),
\qquad
r_\beta=\sum_i \omega_i c_i B_\beta(p_i),$
得到
$(A+\lambda S)x=b+\lambda r.$
这就是 screened Poisson 的典型离散线性系统。

5.6 screened 为什么更稳定
普通 Poisson 只控制梯度：
$\|\nabla\chi-V\|^2.$
如果两个函数只差一个低频偏移，梯度几乎可能一样。
screening 额外加入“函数值要在采样点附近对齐”的约束，能：


抑制低频漂移


更贴近原始点的位置


在稀疏、非均匀采样时更稳


因此实际重建效果通常比纯 Poisson 更贴点。

6. 颜色、法线外的属性如何恢复
你问“点上有颜色等属性信息如何在泊松重建时恢复”，通常有三种路子。

6.1 最常见：几何先重建，属性后插值
先用 Poisson 或 screened Poisson 重建出几何表面网格 $M$，再把点云属性映射到网格顶点或面片。
设点云有属性 $a_i$，比如颜色 $c_i=(r_i,g_i,b_i)$。
对于网格顶点 $v$，找附近点 $p_i$，做加权平均：
$a(v)=\frac{\sum_i w_i(v)a_i}{\sum_i w_i(v)}.$
常用权重：
1）最近邻
$a(v)=a_{i^*},\qquad i^*=\arg\min_i \|v-p_i\|.$
简单但容易噪声大。
2）反距离加权
$w_i(v)=\frac{1}{(\|v-p_i\|^2+\epsilon)^{\gamma/2}}.$
3）带法向一致性的权重
$w_i(v)
=
\exp\!\left(-\frac{\|v-p_i\|^2}{2\sigma_d^2}\right)
\cdot
\max(0,n_v\cdot n_i)^\eta.$
这样可减少跨表面串色。
这是工程上最常用的方法，因为：


实现简单


几何与属性解耦


可对任意属性直接套用：颜色、反射率、温度、语义标签概率等



6.2 直接重建属性场
也可以像重建 $\chi$ 一样，直接重建一个属性函数 $a(x)$。
例如颜色的每个通道 $r(x),g(x),b(x)$ 分别求解一个 screened 插值问题：
$\min_a
\int_\Omega \mu(x)\|\nabla a(x)\|^2 dx
+
\lambda\sum_i \omega_i (a(p_i)-a_i)^2.$
弱形式为
$\int_\Omega \mu \nabla a\cdot\nabla\phi\,dx
+
\lambda\sum_i \omega_i a(p_i)\phi(p_i)
=
\lambda\sum_i \omega_i a_i\phi(p_i).$
离散后：
$(K+\lambda S)\,y=\lambda q.$
其中 $y$ 是属性场系数。
求得 $a_h(x)$ 后，在表面顶点上采样即可。
这种方式更平滑，更像“全局正则化插值”，但代价更高。

6.3 用原始点到隐式表面的投影恢复属性
另一种常见做法：


重建隐式函数 $\chi$


提取等值面网格


对每个网格顶点 $v$，沿法向或最近点搜索找到对应原始点集


聚合这些点的属性


如果网格是从隐式函数的 Marching Cubes 提取的，这种方法也很自然。

6.4 颜色恢复的实用建议
如果你要做高质量颜色恢复，通常建议：


先重建几何


再对顶点做局部加权颜色融合


权重同时考虑：


欧氏距离


法向夹角


可见性/遮挡


采样置信度




比如
$w_i(v)=
\exp\!\left(-\frac{\|v-p_i\|^2}{2\sigma_d^2}\right)
\exp\!\left(-\frac{(1-n_v\cdot n_i)^2}{2\sigma_n^2}\right)
\omega_i.$
对 RGB 三个通道分别平均即可。
若有多视图颜色，还可加“视线夹角”和“图像清晰度权重”。

7. 如何估算重建后的点密度，并裁剪远离点云的网格
这是 Poisson 重建里非常实用的一步。很多实现会给每个顶点一个“density”值，用于裁剪低支持区域。

7.1 密度本质上是什么
你想估计的是：
网格某处是否真的被原始点云充分支持。
换句话说，就是表面附近有多少输入点、这些点离它多近、尺度是否匹配。
因此“密度”通常不是概率密度的严格定义，而是一个采样支持度。

7.2 最直接的密度估计：核密度估计
对表面上的一点 $x$ 定义
$\rho(x)=\sum_i K\!\left(\frac{\|x-p_i\|}{h_i}\right),$
其中 $K$ 是核函数，例如高斯核：
$K(r)=\exp(-r^2/2).$
如果点云非均匀采样，可以给每个点权重 $\omega_i$：
$\rho(x)=\sum_i \omega_i\,K\!\left(\frac{\|x-p_i\|}{h_i}\right).$
在网格上，只需对每个顶点 $v$ 算 $\rho(v)$。

7.3 与 Poisson 层次结构一致的密度
在八叉树 Poisson 重建里，更自然的是用“点对八叉树节点的累积支持”定义密度。
因为 PoissonRecon 本来就把点 splat 到层次基函数上，所以可以同时维护一个“采样权重场”。
例如构造
$d_h(x)=\sum_j d_j B_j(x),$
其中 $d_j$ 由点样本对基函数节点的贡献累加得到。
最后对提取出的网格顶点 $v$ 直接评估 $d_h(v)$，这就是密度值。
这类密度与重建过程使用的尺度一致，因此比单纯最近邻计数更稳。

7.4 顶点密度如何用于裁剪
得到每个网格顶点密度 $\rho(v)$ 后，有几种用法。
方法 1：阈值删顶点/删面
若三角形 $f=(v_1,v_2,v_3)$ 的密度太低：
$\min(\rho(v_1),\rho(v_2),\rho(v_3))<\tau$
则删除该面。
方法 2：连通域筛选
低密度区域往往形成漂浮碎片。可以：


先按密度阈值去掉低支持面


再对剩余网格做连通域分析


删除面积太小的连通块


方法 3：按分位数截断
密度数值尺度会随数据集变化，因此常不用固定阈值，而用分位数，比如保留密度前 $95\%$ 区域，裁掉最低 $5\%$ 顶点附近的面。

7.5 更稳的裁剪准则
只用欧氏邻近有时会误把相邻薄层串起来，所以可加入法向与隐式距离约束：
$\rho(v)=\sum_i
\exp\!\left(-\frac{\|v-p_i\|^2}{2\sigma_d^2}\right)
\max(0,n_v\cdot n_i)^\eta.$
这样背向点贡献更小。
还可以结合隐式函数梯度强度：


若 $|\nabla\chi(v)|$ 异常小，说明局部表面不稳定


若 $\rho(v)$ 也低，则更应裁剪



8. 一个统一的离散模板
其实你问的这些都能放进一个统一模板里。

8.1 纯 Poisson 重建
连续能量：
$E(\chi)=\int_\Omega \|\nabla\chi -V\|^2 dx.$
弱形式：
$(\nabla\chi,\nabla\phi)=(V,\nabla\phi).$
离散系统：
$Ax=b,$
其中
$A_{\alpha\beta}=(\nabla B_\alpha,\nabla B_\beta),
\qquad
b_\beta=(V_h,\nabla B_\beta).$

8.2 Screened Poisson 重建
连续能量：
$E(\chi)=\int_\Omega \|\nabla\chi -V\|^2 dx
+\lambda\sum_i \omega_i (\chi(p_i)-c_i)^2.$
弱形式：
$(\nabla\chi,\nabla\phi)
+\lambda\sum_i \omega_i \chi(p_i)\phi(p_i)
=
(V,\nabla\phi)
+\lambda\sum_i \omega_i c_i\phi(p_i).$
离散系统：
$(A+\lambda S)x=b+\lambda r,$
其中
$S_{\alpha\beta}=\sum_i \omega_i B_\alpha(p_i)B_\beta(p_i),
\qquad
r_\beta=\sum_i \omega_i c_i B_\beta(p_i).$

8.3 属性恢复
对属性 $a$：
简单后处理
$a(v)=\frac{\sum_i w_i(v)a_i}{\sum_i w_i(v)}.$
全局 screened 插值
$(K+\lambda S)y=\lambda q.$

8.4 密度估计
$\rho(v)=\sum_i \omega_i K\!\left(\frac{\|v-p_i\|}{h_i}\right),$
或者用 Poisson 八叉树上的支持场 $d_h(v)$。
然后按 $\rho(v)<\tau$ 裁剪。

9. 你最关心的那个“为什么连续和离散等价”一句话版
严格地说，不是“连续问题和离散问题完全等价”，而是：


连续问题是在无限维函数空间里最小化能量；


离散问题是在有限维子空间 $X_h$ 里最小化同一个能量；


用同样的变分原理，就得到离散线性系统 $Ax=b$。


所以离散系统不是拍脑袋来的，而是：
连续变分问题在有限维空间上的 Galerkin/Ritz 投影。

10. 实际 PoissonRecon 里还要注意的几点


纯 Poisson 常数不定性
通常最终等值面阈值会吸收这部分影响，或通过额外约束固定。


screened 参数 $\lambda$
太小接近普通 Poisson，太大会过拟合点噪声。


法向方向必须尽量一致
否则 $\nabla\cdot V$ 会错误，重建会破裂。


颜色恢复最好和几何分开
几何全局解，颜色局部投影，通常最稳。


密度裁剪非常重要
能去掉远离点云支持的“鼓包”“桥接”“飘片”。

2. 输入点云预处理

2.1 坐标归一化
先求点云包围盒：
$b_{\min}=\min_i p_i,\qquad b_{\max}=\max_i p_i.$
一般会把它嵌入一个立方体域 $\Omega=[0,1]^3$ 或 $[-1,1]^3$，并稍微留边：
$\Omega = \text{expand}\big(\text{bbox}(p_i),\; \eta\big),$
其中 $\eta$ 可取比如 5%–20%。
实现上常做：
$\tilde p_i = \frac{p_i - b_{\min}}{s},
\qquad
s=\max(b_{\max}-b_{\min}),$
然后再平移到 $[0,1]^3$ 中央。
这样做的原因：


八叉树层级尺度统一


最大深度和空间分辨率关系明确


边界条件更好处理



2.2 法向预处理
法向必须至少做到两件事：
单位化
$n_i \leftarrow \frac{n_i}{\|n_i\|}.$
方向一致
Poisson 非常依赖法向方向一致性。若法向乱翻，会导致散度源项正负混乱。
常见方法：


有传感器视点时，让法向朝向相机外/内


无视点时，用 kNN 图传播法向符号


MST / graph propagation


或先做局部 PCA，再全局定向


如果法向方向不一致，后面 splat 到向量场时就已经坏了。

2.3 权重 $w_i$
$w_i$ 一般表示采样可信度、面积权重、置信度，或者局部采样密度修正。
可选几种意义：
情况 A：样本置信度
直接作为法向贡献强度：
$V \sim \sum_i w_i n_i \delta(x-p_i).$
情况 B：面积权重 / 非均匀采样修正
比如点密处不想重复计数太多，可令
$w_i \approx \frac{1}{\rho_i}$
其中 $\rho_i$ 是局部点密度估计。
情况 C：screening 权重
在 screened 项里单独再用一份权重 $\omega_i$，可以与 $w_i$ 相同也可以不同。
实践里你可以先统一用一个 $w_i$，后面分成：


法向项权重 $w_i^{(V)}$


screening 权重 $w_i^{(S)}$


颜色传播权重 $w_i^{(C)}$



2.4 颜色/属性 $c_i$
颜色 $c_i$ 在几何重建阶段通常不直接进入 $\chi$ 的 PDE。
你最好先把它作为附加字段存起来：


point[i].pos = p_i


point[i].normal = n_i


point[i].weight = w_i


point[i].color = c_i


后面有两条路线：


几何重建完后投影到网格


同步重建属性场


你现在写八叉树时，先把点与叶节点/相邻节点的关联存好，后面颜色恢复会很方便。

3. 重建域 $\Omega$ 和八叉树层级

3.1 八叉树域
八叉树根节点对应整个域 $\Omega$，一般是立方体。
根节点记为：


中心 $o_\text{root}$


边长 $h_0$


深度为 $d$ 的节点边长为
$h_d = \frac{h_0}{2^d}.$

3.2 深度的意义
最大深度 $D$ 决定最高空间分辨率。
如果物体包围盒归一化到单位立方体，深度 $D$ 的体素边长是
$h_D = 2^{-D}.$
可理解为：
深度越大，能表达的细节越细，但矩阵规模越大。

4. 自适应八叉树怎么建
这是“点云到八叉树基函数”的第一核心。

4.1 节点数据结构建议
每个八叉树节点至少存：


depth


bbox / center / halfWidth


children[8]


pointIndices 或计数


isLeaf


active（是否参与基函数）


basisIndex（若这个节点对应未知）


邻居/父子指针（方便 later stencil）


可再加：


样本累计权重


法向累计


密度累计


screening 点列表


颜色累计统计



4.2 最基础的细分准则
对一个节点 $q$，若满足以下条件就继续细分：


深度 $< D_{\max}$


节点中点数太多：$N_q > N_{\text{split}}$


或局部几何复杂度高


或节点内法向变化大


或用户要求固定深度


最简单版本：
$\text{split}(q)\iff \big(\#\{p_i\in q\}>N_{\text{split}}\big)\land \big(\text{depth}(q)<D_{\max}\big).$
这样得到一个随点分布自适应的八叉树。

4.3 更适合 Poisson 的细分准则
Poisson 重建不只是“点多就细分”，而是希望：


表面附近细


空域深处可粗


基函数邻域耦合不要太不规则


所以更适合用：
规则 1：样本数量
节点样本数大于阈值就分裂。
规则 2：样本加权密度
$\sum_{p_i\in q} w_i > \tau_w.$
规则 3：法向复杂度
节点内法向方差大，说明局部几何弯曲/交界多，应细分。
规则 4：最小深度
表面包围范围至少到某一层都细分，避免太粗。

4.4 平衡八叉树（非常重要）
实际构建后，通常要做 2:1 balance 或邻层差不超过 1 的平衡化。
即若两个相邻节点共享面/边/角，则深度差不要太大：
$|d(q_1)-d(q_2)|\le 1.$
原因：


基函数重叠和积分 stencil 更稳定


避免非常悬殊的局部尺度导致矩阵装配复杂


Marching Cubes / Dual Contouring 后处理更容易


所以流程通常是：


按点云递归细分得到初始树


再做 balance refinement



4.5 点插入时的实现建议
点插入常见两种做法：
做法 A：每层递归下传
每次 split 时把当前节点的点分配到 8 个子节点中
优点：简单
缺点：点复制和重排较多
做法 B：先排序/编码
用 Morton code / Z-order 对点排序，然后按层批量构树
优点：大规模更快
缺点：实现复杂
你如果是第一次写，建议先用 A。

5. 八叉树上“基函数”到底是什么
这是第二核心。
不是“有了八叉树节点就自动有未知量”，而是你还得定义每个未知量对应的函数 $B_\alpha(x)$。

5.1 最朴素的想法：每节点一个体素常数函数
比如：
$B_q(x)=
\begin{cases}
1,&x\in q\\
0,&\text{otherwise}
\end{cases}$
这不能用来做 Poisson，因为：


梯度是分布，不好算


不连续


拉普拉斯和弱形式都不舒服


所以实际必须用更平滑的局部基。

5.2 常见选择：FEM hat / B-spline / 分片多项式基
在 Poisson 八叉树重建里，最常见的是八叉树上定义的局部支撑、低阶多项式基函数。
Kazhdan 那套实现的核心是 B-spline 风格的层次基函数。
你可以把它理解成：


每个活跃节点 $q$ 或节点中心/角点，对应一个基函数 $B_q(x)$


$B_q$ 只在 $q$ 附近若干个相邻单元上非零


是 $C^0$ 或更高连续的分片多项式


支撑局部，所以矩阵稀疏



5.3 最容易理解的方式：张量积一维基函数
先在 1D 上定义某个基函数 $b(t)$，比如 hat 函数或 B-spline：
例如 hat 函数
$b(t)=\max(1-|t|,0).$
给定某层网格节点位置 $x_j$，对应
$b_j(x)=b\!\left(\frac{x-x_j}{h}\right).$
再在 3D 中做张量积：
$B_{jkl}(x,y,z)=b_j(x)b_k(y)b_l(z).$
这就是一个局部支撑基函数。
八叉树上会更复杂一些，因为不同区域层级不同，但局部思想是一样的。

5.4 在自适应八叉树上的基函数
对自适应树，不能简单用整层规则网格所有基函数，因为很多地方没细分到那层。
常见方法是：


只保留“活跃节点”关联的基函数


基函数支撑跟节点尺度 $h_q$ 对应


在粗细交界区通过 balance 保证函数能兼容拼接


记一个节点 $q$ 的中心为 $o_q$，尺度为 $h_q$，则可以写成类似：
$B_q(x)=b\!\left(\frac{x_x-o_{q,x}}{h_q}\right)
       b\!\left(\frac{x_y-o_{q,y}}{h_q}\right)
       b\!\left(\frac{x_z-o_{q,z}}{h_q}\right).$
注意这只是“形状理解式”。
真正实现时要考虑：


节点是否 active


粗细层交界如何截断/限制


同层/跨层重叠关系



5.5 你到底该选哪类基函数
如果你是第一次写八叉树 Poisson，我建议你在实现上分三档：
档 1：先做最容易验证的
规则网格 + 三线性 hat 基函数
先在固定深度八叉树（其实就是均匀体素）上写通整个流程。
因为这时：


基函数简单


点定位简单


$A,b,S$ 都容易组装


screened、边界、颜色都可先跑通


档 2：再做自适应 + 平衡
保持同样的局部基定义，只是节点不再满树。
档 3：再做真正的 B-spline 风格层次基
这时更接近 Kazhdan 风格。
如果你现在就是要写“真正八叉树版本”，那我下面按“自适应八叉树 + 局部支撑张量积基函数”继续讲，不强依赖具体是 hat 还是 quadratic B-spline。

6. 哪些节点要有基函数：active nodes / FEM nodes

6.1 两种索引观念
这里容易混：
观念 A：节点就是未知量
每个八叉树单元 $q$ 对应一个系数 $x_q$。
观念 B：网格点/控制点才是未知量
单元只是组织结构，未知量在节点角点/中心/控制点上。
Kazhdan 的 B-spline 路线更接近 B。
但从实现理解上，你也可以先把“每个活跃单元有一个基函数”当抽象，不影响主流程。

6.2 活跃节点如何选
通常不会对整棵树所有节点都设未知量。
常见做法：


与采样表面邻近的节点 active


或所有叶节点 + 必要祖先节点 active


或满足支撑重叠需求的 closure 节点 active


一个实用方案：


所有叶节点 active


所有叶节点的祖先 active（方便层次耦合）


所有与 active 节点相邻且支撑重叠的节点也 active


这样能保证矩阵装配时邻接完整。

6.3 为什么要包含祖先/邻居
因为你的基函数不是只在单个叶子上有值，而是在周围若干单元上有支撑。
如果只保留叶节点本身，很多积分需要的相邻函数就找不到。

7. 从点云到“向量场右端项”的构建：法向 splat
这一步是 Poisson 最关键的前处理。
你输入的是离散法向样本 $(p_i,n_i,w_i)$，但方程里需要的是
$b_\beta = \int_\Omega V_h(x)\cdot \nabla B_\beta(x)\,dx.$
所以你要把点样本转换成离散右端。

7.1 连续模型
理论上常把法向样本视为分布型向量场：
$V(x)=\sum_i w_i n_i \,\delta(x-p_i).$
代入右端：
$b_\beta
=
\int_\Omega \left(\sum_i w_i n_i \delta(x-p_i)\right)\cdot\nabla B_\beta(x)\,dx
=
\sum_i w_i n_i\cdot\nabla B_\beta(p_i).$
这就是最重要的实现公式之一。

7.2 实现意义
这说明你根本不必先构造一个显式体向量场 $V_h$ 再去积分。
可以直接按点累加右端：
$b_\beta \mathrel{+}= w_i\, n_i\cdot\nabla B_\beta(p_i).$
所以“法向 splat”本质上就是：

对每个点 $p_i$，找到所有在 $p_i$ 处非零的基函数 $B_\beta$，把
$w_i n_i\cdot\nabla B_\beta(p_i)$
加到对应右端 $b_\beta$ 上。


7.3 一个点会影响多少个基函数
因为 $B_\beta$ 是局部支撑的，所以一个点只影响附近少数基函数。
若你用张量积 hat 基函数，在规则网格里点会影响周围 $2^3=8$ 个节点基函数。
若用更高阶 B-spline，则可能影响 $4^3=64$ 个。
在自适应八叉树里，影响集合还会受粗细层交界影响，但仍然是局部常数规模。

7.4 法向 splat 的实现步骤
对每个点 $i$：


找到包含它的叶节点 $q_i$


找到所有在 $p_i$ 处支撑非零的 active 基函数索引集合 $\mathcal N(p_i)$


对每个 $\beta\in\mathcal N(p_i)$：


计算 $B_\beta(p_i)$（screening 用）


计算 $\nabla B_\beta(p_i)$（Poisson 右端用）




更新：
$b_\beta \mathrel{+}= w_i\, n_i\cdot\nabla B_\beta(p_i)$


你最好把第 2–3 步缓存，因为 screened 和颜色恢复都会复用。

8. screened=true 时，点约束项怎么构建

8.1 screened 的连续模型
screened 项一般是：
$E_{\text{screen}}(\chi)
=
\lambda \sum_i \omega_i \big(\chi(p_i)-c_i^{(\chi)}\big)^2.$
注意这里我把目标值写成 $c_i^{(\chi)}$，因为它不是你的颜色 $c_i$。
这是隐式函数在点上的目标值，比如：


表面点都应满足 $\chi(p_i)=\tau$


常设 $\tau=0$ 或 $\tau=1/2$


所以不要把颜色 $c_i$ 和 screened 的 target value 混了。

8.2 离散后是什么
因为
$\chi_h(p_i)=\sum_\alpha x_\alpha B_\alpha(p_i),$
screening 项变成
$E_{\text{screen}}(x)
=
\lambda \sum_i \omega_i
\left(
\sum_\alpha x_\alpha B_\alpha(p_i)-c_i^{(\chi)}
\right)^2.$
展开后对应矩阵项：
$S_{\alpha\beta}
=
\sum_i \omega_i B_\alpha(p_i)B_\beta(p_i),$
和右端：
$r_\beta
=
\sum_i \omega_i c_i^{(\chi)} B_\beta(p_i).$
最终系统多出
$(A+\lambda S)x = b + \lambda r.$

8.3 实现步骤
与法向 splat 几乎共用邻域搜索：
对每个点 $i$：


找到 $\mathcal N(p_i)$


计算所有 $B_\alpha(p_i)$


对所有 $\alpha,\beta\in \mathcal N(p_i)$：
$S_{\alpha\beta} \mathrel{+}= \omega_i B_\alpha(p_i)B_\beta(p_i)$


对每个 $\beta\in\mathcal N(p_i)$：
$r_\beta \mathrel{+}= \omega_i c_i^{(\chi)} B_\beta(p_i)$



8.4 screened = false
那就跳过 $S$ 和 $r$，只保留 Poisson 项。

9. 颜色 $c_i$ 在“前处理”里怎么安排
你输入里有颜色 $c_i$。它通常不直接进几何 PDE。
但在八叉树构建阶段应同时准备好两种可能路线。

9.1 路线 A：几何后投影颜色
你只需保证：


每个点属于哪个叶节点可查询


每个网格顶点以后能快速查附近点


最好提前为点建 kNN / hash / leaf bucket


这样重建出网格后，可以做：
$C(v)=\frac{\sum_i \theta_i(v)c_i}{\sum_i \theta_i(v)}$
其中 $\theta_i(v)$ 是距离/法向加权。
实现层面上，这意味着八叉树构建时就把点挂在叶节点里，后面很方便局部搜点。

9.2 路线 B：同步重建颜色场
若你希望颜色也通过八叉树基函数重建，则对每个颜色通道 $u\in\{r,g,b\}$ 解：
$\min_a \;
\mu\int_\Omega \|\nabla a\|^2dx
+
\lambda_c \sum_i \omega_i(a(p_i)-c_i^{(u)})^2.$
这会得到另一个系统：
$(K+\lambda_c S)y^{(u)}=\lambda_c q^{(u)}.$
其中 $S$ 可以直接复用 screening 的采样矩阵，只是右端变成颜色通道值。
所以从“前处理”角度说，把每个点在局部基函数上的取值 $B_\alpha(p_i)$ 缓存下来非常有价值。

10. neumann_or_dirichlet = 0/1 时，前处理要怎么准备

10.1 Neumann（自然边界）
如果 neumann_or_dirichlet = 0 表示 Neumann：
那通常你什么都不用在边界上强制固定，只需：


让弱形式自然成立


外边界默认满足
$\frac{\partial\chi}{\partial n}=V\cdot n$
或在实际实现中视为自然边界


前处理影响


边界节点照常建树和建基函数


不需要额外固定边界值


但要知道：系统常常会有常数零空间


所以实现上后面要么：


固定一个自由度


要么加均值约束


要么最后用等值面常数吸收



10.2 Dirichlet（固定边界值）
如果 neumann_or_dirichlet = 1 表示 Dirichlet：
你必须先明确边界值是什么。通常 Poisson 重建里最常见的是：
$\chi=0 \qquad \text{on }\partial\Omega.$
这表示域边界视作“外部”。
前处理影响
你需要识别所有与域边界相接的基函数自由度：


节点中心在边界


或基函数支撑与 $\partial\Omega$ 相交


然后把它们标记为 boundary-fixed。
后面组装矩阵时这些自由度不作为未知，或者被强行置值。

10.3 八叉树上如何识别边界自由度
对一个节点 $q$，若其支撑域 $\text{supp}(B_q)$ 与重建域边界相交，则它可能是边界自由度。
最简单判定：


节点 bbox 贴到根域面


或节点中心离边界小于本层支撑半径



11. 一个你真正该保存的“样本—基函数邻接表”
这是实现上最关键的数据结构之一。

12. 八叉树基函数的一个实用构造视角
你说你正在“写八叉树”，所以我给你一个很适合实现的思路：

12.1 先把每个节点看成一个局部坐标系
对节点 $q$，中心 $o_q$，边长 $h_q$。
定义局部坐标：
$\xi = \frac{x-o_q}{h_q}.$
然后选一个母函数 $b(\xi)$，例如一维 hat 或 B-spline。
则三维基函数是
$B_q(x)=b(\xi_x)b(\xi_y)b(\xi_z).$
梯度就是
$\nabla B_q(x)=
\frac{1}{h_q}
\begin{bmatrix}
b'(\xi_x)b(\xi_y)b(\xi_z)\\
b(\xi_x)b'(\xi_y)b(\xi_z)\\
b(\xi_x)b(\xi_y)b'(\xi_z)
\end{bmatrix}.$
这个公式特别重要，因为点 splat 的右端就靠它。

12.2 自适应树里如何避免粗细层冲突
需要两个机制：
机制 1：平衡树
相邻层级差不超过 1
机制 2：active basis closure
某个点附近如果细层基函数激活，则支撑重叠到的粗层相关基函数也要存在
这样你就不会在求 $B_\alpha(p)$、$\nabla B_\alpha(p)$ 时出现“理论应有但数据结构没有”的情况。

2. 为什么“叶子 + 父节点原始基函数全留”是错的
设 1D 先想象，最容易看清。
假设 coarse level 有一个基函数 $B^0_i$，支撑跨两个粗单元。
现在你把其中一半区域细化到 level 1，于是细层里会出现几个更窄的 $B^1_j$。
由于样条空间嵌套，
$V^0 \subset V^1$
所以粗层函数 $B^0_i$ 本来就能被细层函数线性表示：
$B^0_i = \sum_j c_{ij} B^1_j$
这意味着：


你如果把 $B^0_i$ 和这些 $B^1_j$ 同时原样保留


那它们就在同一块区域里“重复描述同一部分自由度”


这不是“更丰富”，而是冗余。
所以标准做法从来不是“父子同留原始函数”，而是：


粗层函数在被细化区域要移除，或裁剪


只保留它在未细化区域的贡献


这就是 hierarchical spline / truncated hierarchical spline 的基本思想。

3. 正确答案：你要的不是普通 hierarchical basis，而是 THB / 截断层级基
如果你问的是：

我已经有自适应八叉树，接下来到底怎么选激活节点和基函数？

最稳妥的答案是：
用 THB-splines（Truncated Hierarchical B-splines） 的规则
因为它正是为了解决你说的这些问题设计的。
它的逻辑不是：


激活所有叶子和父节点的原始基函数


而是：


在每一层选一批“候选基函数”


然后对粗层基函数，把它在已细化区域内可由细层表示的部分截断掉


最终保留的是“粗层未被细化覆盖部分 + 细层新引入部分”


这样父子层虽都可能“存在”，但不是原始重叠存在，而是粗层被裁剪后再存在。

4. 你到底该怎么选激活节点
下面给你一个真正可执行的规则，不绕。

4.1 先定义层级叶子单元集合
你已经有八叉树。设：


level $0,1,\dots,L$


每个 level 上有一组网格单元


最终叶子单元集合记作 $\mathcal L$


每个叶子单元属于某一层。

4.2 定义每一层的“活动区域”
令 $\Omega^\ell$ 表示 level $\ell$ 上仍由该层负责描述的区域。
标准层级结构满足：
$\Omega^0 \supset \Omega^1 \supset \cdots \supset \Omega^L$
其中：


$\Omega^{\ell+1}$ 是 level $\ell$ 中被细化的区域


也就是被更细叶子覆盖的那部分区域


更直白地说：


level $\ell$ 的某个单元如果不是叶子，说明它被细化了


那它所在区域属于更细层活动区域


反过来，只有未被更细层取代的部分，才该由当前层保留



4.3 level $\ell$ 上哪些基函数候选进入？
对每层的标准 tensor-product B-spline 基 $\mathcal B^\ell$，先选出那些满足：
$\operatorname{supp}(B) \subset \Omega^\ell$
并且
$\operatorname{supp}(B) \not\subset \Omega^{\ell+1}$
的基函数。
这句话什么意思？


它的支撑要落在当前层有效区域内


但不能完全被下一层细化区吞掉


这就是经典 HB-spline 的激活规则。
但注意：这还不够好。
HB 仍可能有较差条件数、局部耦合偏大。真正工程上更推荐再做下一步：

4.4 对这些粗层基函数做 truncation（截断）
对 level $\ell$ 上被保留的每个粗层基函数 $B^\ell$：


先把它用 level $\ell+1$ 的细层基展开：
$B^\ell = \sum_j c_j B^{\ell+1}_j$


对其中那些支撑完全落在细化区 $\Omega^{\ell+1}$ 内的细层项，删掉


保留剩余项，得到截断后的粗层函数
$\operatorname{trunc}^{\ell+1}(B^\ell)$


如果还有更深层，就继续递归截断。
最终得到的是 THB 基。

5. 这时父子层“如何同时运作”？
这是你最关心的问题，我直接说本质：
不是“父层函数 + 子层函数一起原样叠加”
而是
“父层函数只在没被细化接管的区域起作用；子层函数只在细化区内补上更细分辨率”
所以父子层不是重复，而是分工。
你可以把它想成：


粗层负责背景场


细层只在局部加细节


但一旦局部被细层接管，粗层在那里的那部分贡献要被裁掉


这就是为什么你直觉上觉得“原始父子重叠肯定不对”——对，你的直觉是对的。
正确做法就是裁掉粗层在细化区中的那部分。

网格节点索引 $(i,j,k)$

$B^\ell_{ijk}(x,y,z)

其中 $b^\ell_i$ 是 1D B-spline。

2.2 八叉树的作用

2.3 THB 的思想
粗层基函数如果覆盖到已经细化的区域，不能原样保留。

最终得到：


粗层基函数只负责“未细化部分”


细层基函数负责“细化区域”


这样避免父子层原始重叠导致的冗余。

3. 先选 1D 母基函数
为了写清楚，我先用次数 $p$ 的 cardinal B-spline。

3.1 1D B-spline 记号
第 $\ell$ 层步长 $h_\ell$，定义

其中 $\beta_p$ 是标准 cardinal B-spline。

$\operatorname{supp}(b^\ell_i)

5. 激活基函数怎么选：HB 候选 + THB 截断

5.1 先选 HB 候选基函数
对每层 $\ell$，遍历该层规则 B-spline 基 $B^\ell_\alpha$。
记 $\alpha=(i,j,k,\ell)$。
候选规则：


$B^\ell_\alpha$ 的支撑必须与本层有效区域有交


但不能完全被更细层区域吞掉


用集合写：
$\operatorname{supp}(B^\ell_\alpha)\cap \Omega^\ell_{act}\neq \varnothing$
等价地，也可写成：
$\operatorname{supp}(B^\ell_\alpha)\not\subset \Omega^{\ell+1}_{ref}$
也就是：


若一个粗层基函数整个支撑都落入更细区域，那它不该被激活


若它有一部分还在未细化区，那它应保留，但后面会被截断


这一步得到的是 HB 候选基。

5.2 为什么还要截断
即便候选基不完全落进细化区，它的支撑往往仍有一部分进入了细化区。
这时不能原样留着，否则它会和细层函数在细化区重复表达。
所以要做 truncation。

5.3 THB 截断定义
对于 level $\ell$ 的一个候选基 $B^\ell_\alpha$：


把它展开到下一层：
$B^\ell_\alpha = \sum_{\beta\in \mathcal C(\alpha)} c_{\alpha\beta} B^{\ell+1}_\beta$
其中 $\mathcal C(\alpha)$ 是它在 $\ell+1$ 层中的子基函数集合


删除那些支撑完全落在 refined 区域中的细层项


用剩余项替换原粗层函数在细化区中的贡献


递归对更深层继续做
最终得到：
$\widetilde B^\ell_\alpha = \operatorname{Trunc}(B^\ell_\alpha)$
这就是 THB 基函数。

对于 degree $p$ 的 cardinal B-spline，存在标准二尺度关系：
$\beta_p(t)=\sum_m h_m \beta_p(2t-m)$
对应到层级：
$b^\ell_i(x)=\sum_{m=0}^{p+1} h_m\, b^{\ell+1}_{2i+m}(x)$
3D 中：
$B^\ell_{ijk}
=
\sum_{a,b,c} h_a h_b h_c \,
B^{\ell+1}_{2i+a,\;2j+b,\;2k+c}$

8. 基函数如何评估：$B(x)$ 和 $\nabla B(x)$
因为最终 THB 基是“若干规则 B-spline 原子项的线性组合”，所以评估很直接。
若某个 THB 基函数 $\widetilde B_\alpha$ 的展开是
$\widetilde B_\alpha(x)=\sum_{m} c_m B^{\ell_m}_{i_mj_mk_m}(x)$
则：
$\widetilde B_\alpha(x)=\sum_m c_m\, B^{\ell_m}_{i_mj_mk_m}(x)$
$\nabla \widetilde B_\alpha(x)=\sum_m c_m\, \nabla B^{\ell_m}_{i_mj_mk_m}(x)$

8.1 规则 3D B-spline 原子项评估
对原子项：
$B^\ell_{ijk}(x_1,x_2,x_3)
=
b^\ell_i(x_1)b^\ell_j(x_2)b^\ell_k(x_3)$
梯度：
$\nabla B^\ell_{ijk}
=
\begin{bmatrix}
(b^\ell_i)'(x_1)b^\ell_j(x_2)b^\ell_k(x_3)\\
b^\ell_i(x_1)(b^\ell_j)'(x_2)b^\ell_k(x_3)\\
b^\ell_i(x_1)b^\ell_j(x_2)(b^\ell_k)'(x_3)
\end{bmatrix}$
注意如果 $b^\ell_i(x)=\beta_p(x/h_\ell-i)$，那导数带 $1/h_\ell$ 缩放。

9. 如何找“点 $p_i$ 处非零的基函数”
这一步很关键，因为 splat 和 screened 都靠它。

9.1 朴素想法
对每个点遍历所有 THB 基，检查 support 是否包含点。
太慢。

9.2 正确做法：基于八叉树 / 层级桶索引
给每个 THB 基函数存一个 supportBox。
然后把基函数按其 support 覆盖到的叶子单元桶里。
C++BuildLeafToBasisMap(leaves, thbBasisSet):    for each leaf:        leaf.supportingBasis = []    for each basis alpha:        overlappedLeaves = QueryLeavesIntersectingBox(alpha.supportBox)        for leaf in overlappedLeaves:            leaf.supportingBasis.push_back(alpha.id)
这样对点 $p_i$：


找到其所在叶子 leaf_i


候选基函数只需枚举 leaf_i.supportingBasis


再精确评估 $B_\alpha(p_i)$ 和 $\nabla B_\alpha(p_i)$



9.3 点到局部基函数缓存
你最好缓存这个：
C++struct PointBasisSample {    int pointId;    vector<int> basisIds;    vector<double> basisVals;    vector<Vec3> basisGrads;};
构造：
C++BuildPointBasisCache(points, leaves, thbBasisSet):    cache = array[points.size]    for each point i:        leaf = LocateLeaf(points[i].pos)        candidates = leaf.supportingBasis        for alpha in candidates:            val = EvalTHBBasis(thbBasisSet[alpha], points[i].pos)            if abs(val) < eps and norm(grad) < eps: continue            grad = EvalTHBBasisGrad(thbBasisSet[alpha], points[i].pos)            cache[i].basisIds.push_back(alpha)            cache[i].basisVals.push_back(val)            cache[i].basisGrads.push_back(grad)    return cache
后面 Poisson、screened、颜色、密度都复用它。

10. 把点法向 splat 到基函数：Poisson 右端
现在进入最关键公式。

10.1 连续模型
Poisson 重建最常用形式：
$\min_\chi \int_\Omega \|\nabla \chi - V\|^2\,dx$
弱形式：
$\int_\Omega \nabla \chi \cdot \nabla \phi\,dx
=
\int_\Omega V \cdot \nabla \phi\,dx$
如果把点法向视为分布型向量场
$V(x)=\sum_i w_i n_i \delta(x-p_i)$
则右端变成
$b_\beta
=
\int_\Omega V \cdot \nabla B_\beta\,dx
=
\sum_i w_i\, n_i\cdot \nabla B_\beta(p_i)$
这就是 splat 公式。

10.2 组装伪代码
C++AssemblePoissonRHS(points, pointBasisCache, numBasis):    b = ZeroVector(numBasis)    for each point i:        wi = points[i].weight        ni = points[i].normal        for local t in 0..cache[i].basisIds.size-1:            beta = cache[i].basisIds[t]            grad = cache[i].basisGrads[t]            b[beta] += wi * Dot(ni, grad)    return b
这一步只用：


点法向


点权重


基函数梯度



11. 组装刚度矩阵 $A$
矩阵：
$A_{\alpha\beta}
=
\int_\Omega \nabla B_\alpha \cdot \nabla B_\beta\,dx$
因为 THB 基是线性组合，所以：
$A_{\alpha\beta}
=
\int_\Omega
\left(\sum_m c_m \nabla \hat B_m\right)
\cdot
\left(\sum_n d_n \nabla \hat B_n\right)\,dx
=
\sum_{m,n} c_m d_n
\int_\Omega \nabla \hat B_m \cdot \nabla \hat B_n\,dx$
其中 $\hat B_m$ 是规则网格原子基。

11.1 实现策略
路线 A：数值积分
对每对支撑相交的 THB 基，在它们共同覆盖的叶子单元上做 Gaussian 积分。
路线 B：预计算原子刚度，再做线性组合
若原子 B-spline 结构规则，可预存同层/邻层模板。
第一版建议 A，简单直观。

11.2 支撑相交对枚举
用 leaf-to-basis map 很方便：


每个叶子有一组支撑到它的基函数


在该叶子内，两两枚举这些基函数，做局部单元积分


伪代码：
C++AssembleStiffness(leaves, thbBasisSet, numBasis):    A = SparseMatrix(numBasis, numBasis)    for each leaf Q:        localBasis = Q.supportingBasis        for a in 0..localBasis.size-1:            alpha = localBasis[a]            for b in a..localBasis.size-1:                beta = localBasis[b]                val = IntegrateOnCell(Q,                      dot(grad B_alpha, grad B_beta))                A.add(alpha, beta, val)                if alpha != beta:                    A.add(beta, alpha, val)    return A
其中 IntegrateOnCell(Q, ...) 用 Gaussian 积分。

12. Screened Poisson 的组装
如果 screened = true，再加点值约束。

12.1 连续/离散形式
screened 能量常写成：
$E(\chi)=
\int_\Omega \|\nabla\chi -V\|^2 dx
+\lambda \sum_i \omega_i(\chi(p_i)-\tau)^2$
其中 $\tau$ 是目标等值，比如 0。
离散后：
$(A+\lambda S)x = b + \lambda r$
其中
$S_{\alpha\beta}
=
\sum_i \omega_i B_\alpha(p_i)B_\beta(p_i)$
$r_\beta
=
\sum_i \omega_i \tau B_\beta(p_i)$

12.2 组装伪代码
C++AssembleScreening(points, pointBasisCache, numBasis, isoValue):    S = SparseMatrix(numBasis, numBasis)    r = ZeroVector(numBasis)    for each point i:        wi = points[i].weight        ids  = cache[i].basisIds        vals = cache[i].basisVals        for a in 0..ids.size-1:            alpha = ids[a]            va = vals[a]            r[alpha] += wi * isoValue * va            for b in a..ids.size-1:                beta = ids[b]                vb = vals[b]                S.add(alpha, beta, wi * va * vb)                if alpha != beta:                    S.add(beta, alpha, wi * va * vb)    return S, r

13. 边界条件处理
你的参数 neumann_or_dirichlet = 0/1。

13.1 Neumann = 0
自然边界，通常不显式改矩阵。
但纯 Neumann 可能有常数零空间。
实践中常用：


固定一个自由度


或加平均值约束


或最后用等值面选择吸收常数偏移


伪代码：
C++if neumann:    FixOneDOFOrAddMeanConstraint(M, rhs)

13.2 Dirichlet = 1
通常取外边界：
$\chi=0 \quad \text{on } \partial\Omega$
你需要先标记边界基函数。
标记规则
若某 THB 基函数支撑与根域边界相交，则 isBoundary=true。
然后强制：
$x_\alpha = g_\alpha$
通常 $g_\alpha=0$。
伪代码：
C++ApplyDirichlet(M, rhs, thbBasisSet):    for each basis alpha:        if alpha.isBoundary:            value = 0.0            ZeroRowAndColumn(M, alpha.id)            M(alpha.id, alpha.id) = 1            rhs(alpha.id) = value
更标准也可做分块消元。

14. 颜色 $c_i$ 怎么接进去
你输入有颜色 $c_i$。
几何方程里通常不用它。最实用做法：
做法 A：几何重建后投影颜色
提取网格后，对每个网格顶点 $v$，找附近点做加权平均：
$C(v)=\frac{\sum_i \theta_i(v)c_i}{\sum_i \theta_i(v)}$
权重可取：
$\theta_i(v)=
\exp\!\left(-\frac{\|v-p_i\|^2}{2\sigma_d^2}\right)
\max(0,n_v\cdot n_i)^\eta w_i$

做法 B：同步重建颜色场
对每个颜色通道 $u\in\{r,g,b\}$ 解：
$(K+\lambda_c S)y^{(u)} = q^{(u)}$
其中 $S$ 复用 screened 的采样矩阵，右端是
$q_\beta^{(u)}=\sum_i \omega_i c_i^{(u)} B_\beta(p_i)$
这也很适合 THB 基。

15. 密度估计与裁剪
你后面肯定要做这个，所以一并说。
最自然的 THB / octree 版本是：
$\rho_\beta = \sum_i w_i B_\beta(p_i)$
这给出每个基函数上的采样支持强度。
然后对网格顶点 $v$：
$\rho(v)=\sum_\beta \rho_\beta B_\beta(v)$
低 $\rho(v)$ 区域说明离真实点云支持远，可裁剪。
简化版也可直接对网格顶点做核密度估计。

16. 一套完整主程序伪代码
C++ReconstructPoissonTHB(points, leaves, maxLevel, degree, screened, lambda, neumann_or_dirichlet):    // 1. build THB basis    thbBasisSet = BuildTHBBasis(leaves, maxLevel, degree)    // 2. build leaf-to-basis map    BuildLeafToBasisMap(leaves, thbBasisSet)    // 3. cache basis samples at points    pointBasisCache = BuildPointBasisCache(points, leaves, thbBasisSet)    // 4. assemble stiffness    A = AssembleStiffness(leaves, thbBasisSet, thbBasisSet.size())    // 5. assemble poisson rhs from normals    b = AssemblePoissonRHS(points, pointBasisCache, thbBasisSet.size())    // 6. screened term if needed    if screened:        isoValue = 0.0        S, r = AssembleScreening(points, pointBasisCache, thbBasisSet.size(), isoValue)        M = A + lambda * S        rhs = b + lambda * r    else:        M = A        rhs = b    // 7. boundary conditions    if neumann_or_dirichlet == 1:        ApplyDirichlet(M, rhs, thbBasisSet)    else:        FixOneDOFOrAddMeanConstraint(M, rhs)    // 8. solve    x = SolveSparseSystem(M, rhs)    return {thbBasisSet, x}

17. 你真正要注意的 6 个实现难点
17.1 最难的是 truncation，不是八叉树
树和平衡都容易，真正难的是把层级样条空间构造对。
17.2 不要把八叉树节点直接当 THB 基函数
THB 基函数锚在规则层级网格上，树只决定哪些区域细化。
17.3 点 cache 非常重要
basisIds / basisVals / basisGrads 这个缓存几乎喂饱后面所有步骤。
17.4 刚度积分先用高斯积分
不要一开始就想闭式模板，先数值积分跑通。
17.5 Screened 的 target 不是颜色
是隐式函数表面等值 $\tau$。
17.6 先 1D/2D 验证，再上 3D
特别是 truncation 和 partition of unity，先做单元测试。
